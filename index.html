Dicechanics: A statistical library

Repo

Docs

Description

Dicechanics is a statistical library for modelling dice mechanics. It calculates the exact probability pr. instruction, and thus doesn't use any random generation. Through its interface one can define custom operations on a set of multiple dice, such that various dice mechanics one can think up can be modelled.

The target audience is boardgames enthusiasts and -designers.

Goals

  • To provide and easy to use interface, such that one can quickly test dice configurations and mechanics
  • Calculating complex dice operations should not take longer to complete than getting a cup of coffee.

Challenges

  • Finding an algorithm that allows one to choose a subset of dice which to do operations on.

    To illustrate what is meant here we can look at an example. You roll five d6 and get [1,1,3,5,6], you then want to sum the 3 largest numbers, which here results in 3+5+6 = 14.

  • Implementing a clean interface full of features, while still having good performance.

Reflection

The library is atm feature complete and can do the various standard operations which it should do. The interface is also quite intuitive although it does have some quirks suck as Die.count having to clean way of defining a range of numbers to count.

The performance on the selective algorithm (the one described in Challenges), is slower than i want it to be however the reason for this is known, and its due to the current caching not being able to recognize already solved sub problems.

I am satisfied with where the library is at, however i am far from done with it and have quite a few ideas for improvement on all fronts.

Future ideas

  • I want to reimplement the library in C, and then extend python with that implementation instead. Weather or Not the binding will be written through the Python API or by the library Pybind is not decided yet.

  • Through the library you can use a die as a decorator to apply a function as a map, thus creating new die. However the only used argument for the decorated function is outcome. Here i want attributes such as the maximum face of the die to be available. Here one can use the python function signature to get the argument names. From which we can "type check" the argument names to ensure only allowed names are given, and if not raise an exception.

PyPreCore: Post-processing Dust simulation

Repo

Thesis

Description

This program was written for my thesis, which i worked on for a year doing covid shutdown. Its a post-processing dust particle simulation, for adding a layer of dust to already executed gas simulations. In this project it was a prestellar core, which is a part of the creation process of stars and planets.

The program was made to run on lagrangian meshes, however the data is not available to me anymore. It can however run on a Bonnor-Ebert sphere, which the code can generate.

Simple maze generator

Repo

Description

This was a small project made over a weekend. It generates a random maze, solves it and then displays it to the console.

This project illustrates how i organize my C projects. I use abstract data types represented by opaque pointers. Two examples of such are the Stack (pointer stack) and the Valstack (value stack). They have an defined interface for which to interact with it, thus hides the data from the user. This naturally leads to more safe code.
